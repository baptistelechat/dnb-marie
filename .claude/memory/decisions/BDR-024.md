## Contexte

Marie avait trouvé un cheat dans l'onglet Association : après chaque bonne réponse, `updateBoardAfterMatch` remplaçait le code matché au même index dans `leftOrder` ET `rightOrder`. La nouvelle paire (pays + capitale, ou région + chef-lieu) apparaissait donc toujours à la même ligne visuelle dans les deux colonnes. Marie pouvait cliquer en boucle sur la même ligne sans jamais lire les labels.

## Décision

Dans `updateBoardAfterMatch` (`AssociationTab/utils.ts`) :

- `leftOrder` : replace-in-place conservé — le nouveau pays hérite de la position de l'ancien, pas de déplacement visuel pour les autres items
- `rightOrder` : replace-in-place puis swap avec un index aléatoire différent du slot matché — la nouvelle capitale atterrit à une ligne imprévisible

Fix dans un seul fichier ; couvre les deux onglets (EU + France) car `FranceAssociationTab/utils.ts` réexporte `updateBoardAfterMatch` depuis `AssociationTab/utils.ts`.

## Conséquences

- Le cheat positionnel est supprimé : Marie doit lire où la capitale a atterri
- La colonne gauche reste stable (zéro déplacement des autres items)
- La colonne droite a au plus 2 items qui changent de position (swap), pas de réorganisation complète
