# BDR-041 — `HistoryDatesAssociationTab` : mapping event→country, date→capital dans AssociationPair

| Décision                                                                                                                                                        | Pourquoi                                                                                                                           | Alternatives considérées                                                                        | Statut |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- | ------ |
| Mapper `id`→`countryCode`, `event`→`country`, `date`→`capital` dans `AssociationPair` pour réutiliser le pattern `FranceAssociationTab` sans modifier les types | `AssociationPair` est générique (3 strings) — le sémantique des champs importe peu tant que `countryCode` est l'identifiant unique | Créer un type `DateAssociationPair` dédié → surcoût inutile, types existants couvrent le besoin | actif  |

Labels résultants : `leftLabel="Événements"` / `rightLabel="Dates"`. Clé leaderboard : `"dnb-histoire-dates-association-leaderboard"`.
