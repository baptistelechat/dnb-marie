# BDR-044 — Phase 6 : toggle direction `date-to-event` / `event-to-date`

| Décision                                                                                                                                                                           | Pourquoi                                                                                                                          | Alternatives considérées                                                     | Statut |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- | ------ |
| Deux modes de jeu distincts — date→event (montrer la date, retrouver l'événement) et event→date (montrer l'événement, retrouver la date) — avec leaderboards séparés par direction | Les deux sens de mémorisation sont pédagogiquement différents ; stockage séparé évite la pollution inter-modes dans le classement | Un seul mode — révision incomplète ; score unique mixte — injuste à comparer | actif  |

`direction: HistoryDatesFlashcardDirection` stocké dans l'entrée leaderboard pour affichage dans la modal et le classement (icône 📅 / 📖). Changement de direction → `resetState()` + nouveau shuffle + rechargement leaderboard filtré.
