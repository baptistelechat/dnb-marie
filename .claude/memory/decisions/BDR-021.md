# BDR-021 — AssociationTab : `AssociationBoardState` avec pool + display + leftOrder/rightOrder indépendants

| Décision                                                                                                                                                                                                                                                                                           | Pourquoi                                                                                                                                                                                  | Alternatives considérées                                                                                                                    | Statut |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| L'état du plateau est modélisé en 4 champs : `pool` (paires en attente), `display` (5 paires affichées), `leftOrder` (ordre colonne gauche), `rightOrder` (ordre colonne droite) — `leftOrder` et `rightOrder` sont des tableaux des mêmes `countryCode` dans des ordres aléatoires **différents** | Séparer données (`display`) et présentation (`leftOrder`/`rightOrder`) permet de brasser les colonnes indépendamment sans dupliquer les données, et de remplacer en place lors d'un match | Un seul tableau `pairs` avec index gauche/droite déduit (couplage fort, swap difficile) ; dupliquer `display` en deux tableaux (redondance) | actif  |

## Structure

```typescript
interface AssociationBoardState {
  pool: AssociationPair[]; // paires encore en attente
  display: AssociationPair[]; // 5 paires actuellement affichées
  leftOrder: string[]; // countryCode colonne gauche — ordre indépendant
  rightOrder: string[]; // countryCode colonne droite — ordre indépendant
}
```

`initBoard()` : shuffle global des 27 paires → display = 5 premières, pool = reste. leftOrder et rightOrder = `shuffle([...codes])` indépendants.
