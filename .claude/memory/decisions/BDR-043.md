# BDR-043 — `normalizeDateAnswer` : `datesMatch` avec mois FR, `/`, 0-padding, années 2 chiffres

| Décision                                                                                                                                                                                               | Pourquoi                                                                                                                           | Alternatives considérées                                                        | Statut |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------- | ------ |
| Créer `datesMatch(input, expected)` dans `normalizeDateAnswer.ts` — chaîne via `normalizeAnswer`, puis remplacement `/`→espace, mapping mois FR→nombre, `parseInt` pour zéros, expansion `XX` → `19XX` | Les élèves saisissent "8 mai 45" / "mai/1945" / "08/05/1945" — la même date a trop de formats valides pour une comparaison directe | Regex ad-hoc par date — non maintenable ; validation back-end — projet statique | actif  |

Expansion 2 chiffres : uniquement si 3 tokens et dernier token est exactement 2 chiffres → `19XX`. Couvre le corpus DNB (dates post-1900). Ne couvre pas les dates antérieures à 1900 — non présentes dans `historicalDates.ts`.
