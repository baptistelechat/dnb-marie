# BDR-011 — Timer démarre au premier submit (pas à l'activation du mode dirigé)

| Décision                                                                                                                                                     | Pourquoi                                                                                                                                   | Alternatives considérées                                                                    | Statut |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------- | ------ |
| `timerStartedAt` est fixé dans `handleSubmit` et `handleSkip` au premier appel — si `timerStartedAt === null`. Pas au moment où le mode "dirigé" est activé. | Donner le temps de lire la carte et de s'orienter avant de chronomètre. L'activation du mode est une intention, pas le début de la partie. | Démarrer le timer dès `handleModeChange("directed")` → pénalise la consultation de la carte | actif  |

## Implémentation

```ts
// Dans handleSubmit et handleSkip
if (mode === "directed" && timerStartedAt === null) {
  setTimerStartedAt(Date.now());
}
```
