# BDR-012 — Détection de fin de partie dans handleNext, pas dans useEffect

| Décision                                                                                                                                                        | Pourquoi                                                                                                                                                                                                         | Alternatives considérées                                                                                                      | Statut |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- | ------ |
| Quand `queue.length === 1` et `lastResult === "correct"`, `setTimerStoppedAt(Date.now())` et `setShowGameOver(true)` sont appelés directement dans `handleNext` | ESLint `react-hooks/set-state-in-effect` interdit les setState directs dans useEffect. Placer la logique dans l'event handler qui provoque la transition est sémantiquement plus juste et évite le double-render | useEffect sur `[queue.length, answeredCodes.size]` → setState interdit dedans, 3ème occurrence du même blocker dans ce projet | actif  |

## Implémentation

```ts
const handleNext = useCallback(() => {
  if (mode === "directed") {
    if (lastResult === "correct") {
      if (queue.length === 1) {
        // dernier pays
        setTimerStoppedAt(Date.now());
        setShowGameOver(true);
      }
      setQueue((prev) => prev.slice(1));
    } else {
      // requeue aléatoire...
    }
  }
}, [mode, lastResult, queue.length]);
```
