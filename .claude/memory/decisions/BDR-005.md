# BDR-005 — Geo data : fetch CDN world-atlas@2 + filtre client (vs fichier local)

| Décision                                | Pourquoi                                                                          | Alternatives considérées                                       | Statut |
| --------------------------------------- | --------------------------------------------------------------------------------- | -------------------------------------------------------------- | ------ |
| Charger world-atlas depuis CDN jsDelivr | react-simple-maps gère le fetch auto via prop `geography` — zéro code additionnel | Fichier local `public/geo/europe-eu.json` (~40 kB, plus léger) | actif  |

## URL utilisée

```
https://cdn.jsdelivr.net/npm/world-atlas@2/countries-50m.json
```

756 kB raw / ~180 kB gzip — chargé une seule fois, mis en cache navigateur.

## Déclencheur de migration vers local

Si la latence CDN devient un problème (usage offline, LAN lent) :

1. Télécharger et filtrer le fichier aux 27 codes ISO numériques UE
2. Placer dans `public/geo/europe-eu.json`
3. Mettre à jour `WORLD_ATLAS_URL` dans `constants.ts`
