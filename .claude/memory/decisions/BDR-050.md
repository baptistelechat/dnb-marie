# BDR-050 — `hintUsed: boolean` dans `onSubmit` vs état parent

| Décision                                                                                                                                                        | Pourquoi                                                                                                                                                                                                                                                        | Alternatives considérées                                                                                                                                                         | Statut |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| `hintRevealed` local à l'AnswerInput · communiqué via `onSubmit(answer, hintUsed: boolean)` · pas de prop `onHintUsed` ni `openedHintCodes: Set` dans le parent | L'état "indice utilisé" est naturellement local au composant qui gère la saisie. Le parent n'en a besoin qu'au moment du submit → le callback existant suffit, aucun prop supplémentaire. Pattern établi dans `DateAnswerInput` comme référence de la codebase. | `openedHintCodes: Set<string>` en state parent + prop `onHintUsed` callbacks — abandonné lors de Phase 10 : trop de props, état extrait sans raison depuis son contexte naturel. | actif  |
