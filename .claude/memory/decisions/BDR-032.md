# BDR-032 — `handleHistorySubjectChange` : reset atomique historySubject + historyTab dans le même handler

| Décision                                                                                                                                                                            | Contexte                                                                                                                             | Alternatives écartées                                                                                                                                    | Statut |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| Quand `historySubject` change, `historyTab` est réinitialisé à la valeur par défaut du nouveau sujet dans le même handler `handleHistorySubjectChange`, en un seul `setState` batch | L'onglet actif est couplé au sujet — chaque sujet a ses propres onglets valides (`HISTORY_TABS_DATES` vs `HISTORY_TABS_PERSONNAGES`) | `useEffect` sur `historySubject` pour réinitialiser `historyTab` : interdit par la règle `react-hooks/exhaustive-deps` et source de double-rendu inutile | actif  |

## Règle appliquée

Même pattern que [BDR-012](BDR-012.md) : les mutations d'état couplées se font dans le handler de l'événement déclencheur, jamais dans un `useEffect`. Un seul rendu, zéro race condition.

## Implémentation

```ts
const handleHistorySubjectChange = (s: HistorySubject) => {
  setHistorySubject(s);
  setHistoryTab(
    s === "dates" ? HISTORY_TABS_DATES[0].id : HISTORY_TABS_PERSONNAGES[0].id,
  );
};
```
