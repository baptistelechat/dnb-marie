# BDR-014 — `.claude/launch.json` configuré avec `npx pnpm run dev` (pnpm absent PATH subprocess)

| Décision | Pourquoi | Alternatives considérées | Statut |
| --- | --- | --- | --- |
| `runtimeExecutable: "npx"` + `runtimeArgs: ["pnpm", "run", "dev"]` dans `.claude/launch.json` | Le binaire `pnpm` n'est pas dans le PATH du subprocess lancé par `preview_start` — ENOENT avec `runtimeExecutable: "pnpm"`. `npx` est dans le PATH standard Node.js et peut résoudre `pnpm` via le registre npm | `runtimeExecutable: "pnpm"` (ENOENT), `runtimeExecutable: "npm"` + `runtimeArgs: ["run", "dev"]` (incompatible avec workspace pnpm) | actif |

## Configuration

```json
{
  "version": "0.0.1",
  "configurations": [
    {
      "name": "dev",
      "runtimeExecutable": "npx",
      "runtimeArgs": ["pnpm", "run", "dev"],
      "port": 5173
    }
  ]
}
```
