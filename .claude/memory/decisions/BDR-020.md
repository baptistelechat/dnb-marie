# BDR-020 — `normalizeAnswer` étendu aux variantes unicode mobiles

| Décision                                                                                                                                                                          | Pourquoi                                                                                                                                                                     | Alternatives considérées                                                                       | Statut |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------- | ------ |
| Le character class de remplacement dans `normalizeAnswer` couvre maintenant 7 caractères : U+0027 `'`, U+2018 `'`, U+2019 `'`, U+2013 `–`, U+2014 `—`, U+0060 `` ` ``, U+002D `-` | iOS/Android substituent silencieusement U+0027→U+2019 (apostrophe) et U+002D→U+2013 (en-dash) lors de la frappe rapide — les réponses correctes étaient rejetées en pratique | Normalisation via `.replaceAll()` chaîné (moins maintenable) ; pas de fix (mauvaise UX mobile) | actif  |

## Regex finale

```ts
.replace(/['''–—`-]/g, " ")
//        ↑U+0027  ↑U+2018 ↑U+2019 ↑U+2013 ↑U+2014
```

> Vérification via `String.fromCharCode()` dans un `.mjs` Node.js — les outils shell (heredoc, node -e) peuvent silencieusement mal encoder les caractères unicode non-ASCII.
