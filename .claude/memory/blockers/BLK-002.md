# BLK-002 — dev-browser Playwright require() : chemin introuvable, 3+ tentatives

| Friction                                                             | Cause réelle                                                                                         | Solution                                                        | Statut |
| -------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- | --------------------------------------------------------------- | ------ |
| `Cannot find module 'playwright'` lors d'un script Playwright direct | `playwright` n'est pas dans le `node_modules` global — il est dans le dossier interne de dev-browser | `require("C:/Users/ASUS/.dev-browser/node_modules/playwright")` | résolu |

## Tentatives

1. `require('playwright')` → module not found
2. `require('playwright-core')` → module not found
3. `require('/root/.dev-browser/node_modules/playwright')` → chemin Unix incorrect sur Windows
4. ✅ `require('C:/Users/ASUS/.dev-browser/node_modules/playwright')` → succès

## Note

Le daemon dev-browser (`daemon.mjs`) gère l'accès à Playwright en interne. Pour des scripts ad hoc en dehors du daemon, utiliser le chemin absolu Windows ci-dessus.
