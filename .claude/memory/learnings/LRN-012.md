# LRN-012 — Bouton GO! comme garde de démarrage explicite du chrono

| Pattern observé                                                                                                                                                                                                                                                               | Contexte                                                                                                                  | Réutilisable ?                                                    |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| `timerStartedAt === null` sépare l'état "prêt" (mode activé) de l'état "en cours" (timer lancé). Le bouton GO! remplace le composant interactif tant que `timerStartedAt === null`. Clic GO → `setTimerStartedAt(Date.now())` → composant interactif affiché + timer visible. | Ajout bouton GO! dans les 4 onglets quiz dnb-marie (MapQuizTab, FranceMapQuizTab, CapitalsQuizTab, FranceCapitalsQuizTab) | Tout quiz chronométré en React avec distinction "prêt / en cours" |

## Implémentation

```tsx
const handleGo = useCallback(() => {
  setTimerStartedAt(Date.now());
}, []);

// JSX
{timerStartedAt === null ? (
  <div>
    <button onClick={handleGo}>GO !</button>
  </div>
) : (
  <QuizPanel ... />
)}
```

## Avantage

Empêche le démarrage accidentel du chrono (anciennement déclenché au premier submit/skip). L'utilisateur peut consulter la carte ou les flashcards sans être chronométré tant qu'il n'a pas cliqué GO.
