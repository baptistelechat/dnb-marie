# LRN-013 — Variante sujet d'un composant : auditer les sous-composants pour labels et valeurs hardcodés

| Pattern observé                                                                                                                                                                                                                                                                   | Contexte                                                                                                                                                                                                                               | Réutilisable ?                                                                |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| Avant de créer une variante sujet d'un composant React, descendre dans l'arbre et repérer les labels et valeurs hardcodés dans les sous-composants — ils sont invisibles depuis l'index et bloquent la réutilisation. Les convertir en props optionnelles avec valeur par défaut. | Création de `FranceAssociationTab` : `AssociationBoard` avait `"Pays"`/`"Capitales"` hardcodés, `AssociationGameOverModal` avait `/27` hardcodé à deux endroits. Ni l'un ni l'autre n'était visible depuis `AssociationTab/index.tsx`. | Tout composant React dont on veut créer une variante (sujet, langue, domaine) |

## Technique

```tsx
// Avant — hardcodé dans le sous-composant
<p>Pays</p>
{firstTryScore} / 27

// Après — prop optionnelle avec défaut
interface Props { leftLabel?: string; totalCountries: number; }
const Comp = ({ leftLabel = "Pays", totalCountries }: Props) => (
  <>
    <p>{leftLabel}</p>
    {firstTryScore} / {totalCountries}
  </>
);
```

## Checklist à passer avant de dériver

1. Grep `hardcoded` strings dans tous les sous-composants du dossier
2. Chercher les nombres "magiques" liés au total du dataset (`/27`, `/13`, etc.)
3. Chercher les labels de colonnes, entêtes de section, textes descriptifs
