# LRN-011 — `flex-1 min-h-0` dans flex-col overflow-auto → compression sous hauteur naturelle + overlap visuel

| Pattern observé                                                                                                                                                                                                                            | Contexte                                                                                                                                                             | Application future                                                                                                                             |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| Dans un flex-col `overflow-auto` de hauteur contrainte, un item `flex-1 min-h-0` peut être comprimé en dessous de la hauteur naturelle de son contenu → le contenu déborde visuellement sans clipping → overlap apparent avec les siblings | Colonne gauche (carte SVG) de MapQuizTab/FranceMapQuizTab à mobile — sidebar `shrink-0` prend ~400px, laissant ~100px à la colonne carte qui devrait en faire ~390px | Tout layout flex-col avec un item de contenu à hauteur variable (image, SVG, carte) et un sibling `shrink-0` dans un conteneur de hauteur fixe |

## Détails

**Mécanisme** : L'algo flex calcule la taille des items dans l'espace disponible. Si `sibling.shrinkTo0 = false` (shrink-0) prend 400px et l'espace total est 580px, l'item `flex-1 min-h-0` reçoit 180px. `min-h-0` lui permet de se comprimer jusqu'à 0 — il accepte 180px même si son contenu (SVG de carte) fait 390px. Le SVG déborde sans être clippé et se superpose visuellement à la sidebar positionnée à y=180px.

**Fix** : retirer `flex-1` et `min-h-0` au breakpoint mobile via Tailwind responsive prefix :

```tsx
// Avant (overlap)
<div className="flex-1 min-w-0 min-h-0 flex flex-col gap-3">

// Après (correct)
<div className="lg:flex-1 min-w-0 lg:min-h-0 flex flex-col gap-3">
```

À mobile : la colonne prend sa hauteur naturelle (carte pleine hauteur), le container parent scrolle pour afficher la sidebar.
À desktop (lg+) : `flex-1 min-h-0` s'applique, la carte remplit l'espace disponible.
