# LRN-027 — Tooltip dans enfant `overflow:hidden` → état `hoveredId` dans le parent

| Pattern observé                                                                                                                                                                                                                                                              | Contexte                                                                                                          | Application future                                                                                                      |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| Si un composant enfant a `overflow: hidden` et doit afficher un tooltip, remonter l'état hover dans le parent (`hoveredId`), passer des callbacks simples (`onHoverStart`, `onHoverEnd`) à l'enfant, et rendre le tooltip dans le parent (contexte de positionnement ouvert) | DropZone range (overflow:hidden pour contraindre le texte vertical) — tooltip clipé quand rendu dans le composant | Tout composant avec overflow:hidden devant afficher un élément débordant ses bornes (tooltip, dropdown, badge flottant) |
