# LRN-010 — Onglets avec scroll hétérogène — appliquer `h-dvh flex-col` conditionnellement par onglet, jamais globalement

| Pattern observé                                                                                                                                                                       | Contexte                                                                                                               | Application future                                                                               |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| Appliquer `h-dvh flex-col` globalement (sur `#root` ou container racine) impose un scroll élément sur tous les onglets, incluant ceux qui n'ont pas besoin d'être viewport-contraints | Carte Quiz dnb-marie — onglet carte devait tenir dans la viewport, Liste et Flashcards devaient scroller naturellement | Toute app React multi-onglets où un onglet est viewport-contraint et les autres sont page-scroll |

## Détails

Quand le container racine a `height: 100dvh` + `display: flex; flex-direction: column`, le `<main>` enfant doit avoir `overflow-auto` pour que son contenu soit accessible. Cela crée une scrollbar élément (OS-rendered) visible sur tous les onglets, y compris ceux où le scroll navigateur natif était souhaité.

**Fix** : conditionner les classes sur l'état `activeTab` :

```tsx
<div className={activeTab === "map-quiz" ? "h-dvh flex flex-col" : "min-h-dvh"}>
  <main className={activeTab === "map-quiz" ? "flex-1 min-h-0 overflow-hidden" : "max-w-2xl mx-auto pb-20"}>
```

Et dans `index.css` :

```css
#root {
  min-height: 100dvh;
} /* pas de display:flex ni height:100dvh */
```
