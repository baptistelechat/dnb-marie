# LRN-002 — react-simple-maps v3 sans types bundlés → créer src/types/react-simple-maps.d.ts

## Pattern observé

`react-simple-maps` v3.0.0 ne contient ni champ `types` ni `typings` dans son `package.json`. TypeScript remonte **TS7016** au build, et les paramètres des callbacks (`geographies`, `geo`) ont un type `any` implicite.

## Fix appliqué

Créer `src/types/react-simple-maps.d.ts` avec les interfaces minimales :

```typescript
declare module "react-simple-maps" {
  import type { CSSProperties, ReactNode } from "react";

  interface GeoObject {
    rsmKey: string;
    id: string;
    properties: Record<string, unknown>;
    geometry: unknown;
  }
  interface GeoStyle {
    fill?: string;
    stroke?: string;
    strokeWidth?: number;
    opacity?: number;
    outline?: string;
    animation?: string;
    cursor?: string;
  }

  interface ComposableMapProps {
    projection?: string;
    projectionConfig?: Record<string, number | string | number[]>;
    width?: number;
    height?: number;
    style?: CSSProperties;
    children?: ReactNode;
  }
  interface GeographiesProps {
    geography: string;
    children: (props: { geographies: GeoObject[] }) => ReactNode;
  }
  interface GeographyProps {
    geography: GeoObject;
    onClick?: () => void;
    style?: { default?: GeoStyle; hover?: GeoStyle; pressed?: GeoStyle };
    "aria-label"?: string;
    role?: string;
    tabIndex?: number;
  }

  export const ComposableMap: (props: ComposableMapProps) => ReactNode;
  export const Geographies: (props: GeographiesProps) => ReactNode;
  export const Geography: (props: GeographyProps) => ReactNode;
}
```

## À retenir

- `@types/react-simple-maps` n'existe pas sur npm (ne pas chercher à l'installer)
- Ce fichier est à jour pour l'usage dnb-marie — adapter si d'autres composants sont utilisés
