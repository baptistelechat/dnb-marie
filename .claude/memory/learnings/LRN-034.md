# LRN-034 — `Map<id, T[]>` pour variantes mélangées par item, reset au replay

| Pattern observé                                                                                                                                                               | Contexte                                                                                                                                                         | Application future                                                                                                                                               |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Stocker les variantes mélangées dans `Map<id, T[]>` en state, réinitialisée à chaque replay via `setShuffledClues(new Map(items.map(f => [f.id, shuffle([...f.variants])])))` | `QuiSuisJeTab` : 5 clues par personnage, ordre différent à chaque partie. `shuffledClues.get(activeId)` fournit les clues dans l'ordre shufflé du round courant. | Tout jeu avec items ayant des variantes ordonnées (indices, étapes, sous-questions) — ne pas mélanger les données source, toujours opérer sur une copie en state |
