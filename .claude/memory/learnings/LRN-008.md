# LRN-008 — `useCallback` dans les custom hooks pour stabiliser les refs exposées

## Pattern observé

Quand un custom hook expose des fonctions utilisées dans les `useCallback` des composants consommateurs, les envelopper dans `useCallback` dans le hook. Sinon, chaque render du hook recrée une nouvelle référence de fonction, ce qui force le recalcul de tous les `useCallback` qui en dépendent — potentiellement des boucles infinies ou des renders excessifs.

## Exemple

```ts
// hooks/useLeaderboard.ts
const getEntries = useCallback((): LeaderboardEntry[] => {
  try {
    return JSON.parse(
      localStorage.getItem(STORAGE_KEY) ?? "[]",
    ) as LeaderboardEntry[];
  } catch {
    return [];
  }
}, []); // deps vides = stable pour toute la vie du hook

const addEntry = useCallback(
  (entry: LeaderboardEntry): LeaderboardEntry[] => {

  [getEntries],

```

```ts
// composant consommateur — stable car addEntry est stable
const handleSave = useCallback((name: string, date: string) => {
  addEntry({ ... });

```

## Règle

Tout custom hook qui expose des fonctions → les envelopper dans `useCallback`. Les deps vides `[]` conviennent pour les fonctions sans closure sur du state React (ex. lecture localStorage).
