# LRN-036 — `useEffect + setTimeout` : `let id | undefined` + `return () => clearTimeout(id)`

| Pattern observé                                                                                                                                                                                                           | Contexte                                                                                                            | Application future                                                                                                                                                    |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Tout `setTimeout` dans un `useEffect` doit déclarer `let id: ReturnType<typeof setTimeout> \| undefined` avant le bloc conditionnel, assigner `id = setTimeout(...)` à l'intérieur, et retourner `() => clearTimeout(id)` | 4 occurrences dans les effets confetti de complétion (EuropTab, FranceTab, FriseLectureTab, HistoirePersonnagesTab) | Appliquer systématiquement dès qu'un setTimeout apparaît dans un useEffect — même si la condition rend le timeout optionnel, clearTimeout(undefined) est un no-op sûr |
