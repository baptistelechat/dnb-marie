# LRN-005 — Diagnostiquer les IDs d'un fichier JSON distant avec node -e fetch

## Pattern observé

Pour investiguer ce que contient un fichier JSON/TopoJSON distant (CDN, API) sans instrumenter le code React et sans l'enregistrer localement :

```bash
node -e "
fetch('https://cdn.jsdelivr.net/npm/world-atlas@2/countries-50m.json')
  .then(r => r.json())
  .then(data => {
    const features = data.objects.countries.geometries;
    const ids = [...new Set(features.map(f => String(f.id)))].sort();
    console.log('IDs présents:', ids.slice(0, 20));
    const target = ['040', '056'];
    console.log('Cibles trouvées:', target.filter(id => ids.includes(id)));
  })
"
```

## Quand l'utiliser

- Bug d'ID non matchant dans une map locale vs données distantes
- Vérifier le format exact (string vs number, zero-padding, casing)
- Inspecter la structure d'un TopoJSON sans le télécharger
