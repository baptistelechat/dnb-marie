# LRN-009 — Réduire EU_COUNTRIES à N éléments pour valider la logique de fin de partie

## Pattern observé

Pour tester une logique de fin de partie (timer stop, modal game-over, scoring) sans jouer l'intégralité d'une session à 27 pays : commenter temporairement `euCountries.ts` pour ne garder que 3–5 éléments, valider le comportement, puis restaurer.

## Procédure

```ts
// src/data/euCountries.ts — temporaire pour test
export const EU_COUNTRIES: Country[] = [
  { code: "DE", name: "Allemagne", capital: "Berlin" },
  { code: "AT", name: "Autriche", capital: "Vienne" },
  { code: "BE", name: "Belgique", capital: "Bruxelles" },
  { code: "BG", name: "Bulgarie", capital: "Sofia" },
  { code: "CY", name: "Chypre", capital: "Nicosie" },
  // ← 22 autres pays commentés
];
```

## Règle

Applicable dès que la validation complète nécessite de jouer >10 étapes. Toujours restaurer les données réelles après validation, vérifier via lint+build que rien n'est cassé.
