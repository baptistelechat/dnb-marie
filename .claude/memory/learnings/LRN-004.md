# LRN-004 — SVG z-order = ordre de rendu DOM : rendre les petits pays en dernier

## Pattern observé

En SVG, les éléments rendus **après** les autres sont **par-dessus** (z-index implicite = ordre DOM). Dans une carte avec `react-simple-maps`, si les grandes géographies (pays non-EU) sont rendues après les petites (Belgique, Luxembourg…), les grandes peuvent couvrir les petites et intercepter les clics.

## Fix appliqué

Double passe de rendu dans `<Geographies>` :

```tsx
<>
  {/* Passe 1 : pays non-EU (grands, fond) */}
  {geographies.filter(geo => !NUMERIC_TO_ALPHA2[geo.id]).map(...)}

  {/* Passe 2 : pays EU (par-dessus, cliquables) */}
  {geographies.filter(geo => !!NUMERIC_TO_ALPHA2[geo.id]).map(...)}
</>
```

## Règle générale

> Toujours rendre les éléments SVG interactifs petits EN DERNIER, après les grands éléments de fond.
