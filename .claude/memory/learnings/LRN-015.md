## Observation

Dans l'onglet Association, `leftOrder` et `rightOrder` sont deux tableaux de codes qui pilotent l'ordre de rendu de chaque colonne. Après un match correct, `updateBoardAfterMatch` remplaçait le code matché à la même position dans les deux tableaux (`array.map(c => c === matched ? next : c)`). Résultat : le nouvel item apparaissait à la ligne X dans la colonne gauche ET à la ligne X dans la colonne droite — la nouvelle paire était toujours visuellement alignée.

## Pourquoi ça marche (le cheat)

Les deux colonnes partagent le même index de remplacement. Cliquer sur la ligne X à gauche + ligne X à droite est toujours correct après un match. Le joueur n'a jamais besoin de lire les labels.

## Quand l'appliquer

Tout jeu d'association à deux colonnes indépendantes où les items sont remplacés après un match. Si les deux tableaux d'ordre subissent le même remplacement en-place, l'alignement devient exploitable.

Fix minimal : garder un tableau stable (left, pour ne pas déplacer les items existants) et faire un swap aléatoire dans l'autre (right) après le remplacement. Seuls 2 items changent de position côté droit.
