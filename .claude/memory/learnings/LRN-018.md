# LRN-018 — Wikimedia HEAD → 429 ≠ URL cassée

## Pattern

Wikimedia Commons rate-limite les requêtes HEAD et retourne **429 Too Many Requests** sur certaines URLs d'upload. Ces images se chargent parfaitement en navigateur (GET normal).

**Ne pas traiter un 429 Wikimedia comme une URL invalide.**

## Signaux fiables de cassure

| Code            | Signification       | Action                            |
| --------------- | ------------------- | --------------------------------- |
| `200`           | OK                  | URL valide ✅                     |
| `429`           | Rate-limited        | URL valide, load en navigateur ✅ |
| `404`           | Not found           | URL cassée ❌                     |
| `ERR` / timeout | Réseau / hors ligne | URL suspecte, re-tester           |

## Comment appliquer

- En vérification batch (PowerShell HEAD) : traiter 429 comme 200
- Seuls les 404 et erreurs réseau justifient de chercher un remplacement
- Si doute sur une image retournant 429 : ouvrir l'URL directement dans le navigateur pour confirmation visuelle
