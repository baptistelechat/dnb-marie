# LRN-007 — ESLint `react-hooks/set-state-in-effect` : déplacer les setState vers les event handlers

## Pattern observé

La règle ESLint `react-hooks/set-state-in-effect` bloque les appels `setState` directs à l'intérieur de `useEffect`. C'est la 3ème occurrence dans ce projet. Le fix systématique : identifier quel événement provoque la transition, et y placer le setState plutôt que de surveiller un state dérivé dans un effect.

## Anti-pattern

```ts
// ❌ setState direct dans useEffect — ESLint error
useEffect(() => {
  if (queue.length === 0 && answeredCodes.size === TOTAL) {
    setTimerStoppedAt(Date.now()); // ← interdit
    setShowGameOver(true); // ← interdit
  }
}, [queue.length, answeredCodes.size]);
```

## Pattern correct

```ts
// ✅ setState dans l'event handler qui provoque la transition
const handleNext = useCallback(() => {
  if (lastResult === "correct" && queue.length === 1) {
    setTimerStoppedAt(Date.now());
    setShowGameOver(true);
  }
  setQueue((prev) => prev.slice(1));
}, [lastResult, queue.length]);
```

## Quand appliquer

Dès que `useEffect` surveille un state pour en déclencher un autre — extraire vers l'événement causal. Exception : les effets purement réactifs (animations, DOM) qui n'ont pas d'événement causal identifiable.
