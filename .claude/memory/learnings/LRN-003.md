# LRN-003 — world-atlas@2 stocke les IDs pays en strings zero-paddées à 3 chiffres

## Pattern observé

Les IDs dans `world-atlas@2/countries-50m.json` (et `110m`) sont des **strings zero-paddées à 3 chiffres** correspondant aux codes ISO 3166-1 numériques :

- Autriche (ISO 040) → `"040"` ❌ pas `"40"` ni `40`
- Belgique (ISO 056) → `"056"` ❌ pas `"56"` ni `56`
- France (ISO 250) → `"250"` ✓ (déjà 3 chiffres)

## Vérification rapide

```js
node -e "fetch('https://cdn.jsdelivr.net/npm/world-atlas@2/countries-50m.json')
  .then(r=>r.json())
  .then(d => console.log([...new Set(d.objects.countries.geometries.map(f=>f.id))].sort().slice(0,10)))"
// → [ '004', '008', '010', '012', '024', '031', '032', '036', '040', '044' ]
```

## Règle d'usage

Toujours utiliser des strings zero-paddées à 3 chiffres dans `NUMERIC_TO_ALPHA2` :

```ts
// ❌ Wrong
"40": "AT", "56": "BE"
// ✅ Correct
"040": "AT", "056": "BE"
```
